# Фаза 5: Планирование разработки MVP
---
## Шаг 1: Декомпозиция проекта на задачи
| ID | Задача | Описание |
|:--:|--------|----------|
| T1 | Проектирование БД | Создание ER-диаграммы, схемы таблиц(пользователи, задачи, категории) |
| T2 | Выбор стека технологий | Выбор фреймворков, библиотек, инструментов для бэкенда и фронтенда |
| T3 | Настройка окружения и репозитория | Инициализация Git, настройка CI/CD, создание виртуальных окружений |
| T4 | Разработка модели Пользователя | Создание модели, миграций, валидация для сущности User |
| T5 | Разработка модели Категории | Создание модели Category, связь с User |
| T6 | Разработка модели Задачи | Создание модели Task, связи с User и Category, поля: название, описание, дедлайн, статус |
| T7 | Реализация аутентификации(регистрация/вход) | API для регистрации, входа, JWT-токенов |
| T8 | CRUD для задач(бэкенд) | Создание API endpoints для создания, чтения, обновления, удаления задач |
| T9 | Фильтрация и сортировка задач(бэкенд) | API для фильтрации по категориям, статусу, дате; сортировка |
| T10 | Механизм напоминаний(бэкенд) | Фоновые задачи для отправки уведомлений о дедлайнах |
| T11 | Написание unit-тестов для бэкенда | Покрытие тестами основных моделей и API |
| T12 | Разработка UI-компонентов аутентификации | React/Vue компоненты: форма входа, регистрации, восстановления пароля |
| T13 | Разработка UI списка задач | Компонент для отображения списка задач с пагинацией |
| T14 | Разработка UI формы создания/редактирования задачи | Компонент с полями ввода, выбором категории, даты |
| T15 | Разработка UI фильтров и сортировки | Компоненты фильтров(выпадающие списки, чекбоксы) и кнопки сортировки |
| T16 | Разработка UI уведомлений | Компонент для показа всплывающих уведомлений(тостер) |
| T17 | Интеграция фронтенда с бэкендом | Настройка API-клиента(axios/fetch), подключение компонентов к API |
| T18 | Интеграционное тестирование | End-to-end тесты критических сценариев(создание задачи, фильтрация) |
| T19 | Деплой бэкенда | Развёртывание сервера(Heroku/VPS), настройка домена, SSL |
| T20 | Деплой фронтенда | Сборка и размещение статики(Netlify/Vercel), подключение к API |

* Количество ресурсов каждого типа: BE=1, FE=1, FS=1
* Длительность рабочего дня: 8 часов
* Рабочая неделя: 5 дней(40 часов)
---
3.2 Применённые эвристики
