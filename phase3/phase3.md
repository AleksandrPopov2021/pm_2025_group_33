# Фаза 3: Оценка проекта
---
## Шаг 1: Use Cases
UC1: Регистрация нового пользователя.
* Основной сценарий: Пользователь вводит email, пароль, подтверждает пароль, нажимает "Зарегистрироваться". Система создаёт учётную запись и отправляет письмо для подтверждения email.
* Альтернативный сценарий: Пользователь использует OAuth (Google/GitHub) для регистрации.
* Негативный сценарий: Введённый email уже занят, система сообщает об ошибке.
---
UC2: Вход в систему.
* Основной сценарий: Пользователь вводит email и пароль, нажимает "Войти". Система проверяет данные и предоставляет доступ.
* Альтернативный сценарий: Пользователь использует OAuth.
* Негативный сценарий: Неверные данные, система сообщает об ошибке.
---
UC3: Создание задачи.
* Основной сценарий: Пользователь нажимает "Добавить задачу", вводит название, описание (опционально), выбирает категорию, приоритет, дедлайн, нажимает "Сохранить". Система добавляет задачу в список.
* Альтернативный сценарий: Пользователь создаёт задачу голосовым вводом (распознавание речи).
* Негативный сценарий: Название не введено, система требует заполнить обязательное поле.
---
UC4: Автоматическая категоризация задачи.
* Основной сценарий: При создании задачи пользователь может нажать "Автокатегоризация", система анализирует текст названия и описания, предлагает категорию.
* Альтернативный сценарий: Пользователь подтверждает предложенную категорию или выбирает другую.
* Негативный сценарий: Система не может определить категорию, оставляет поле пустым или предлагает "Другое".
---
UC5: Просмотр списка задач с фильтрацией и сортировкой.
* Основной сценарий: Пользователь открывает список задач, видит все задачи. Может применить фильтры (по категории, статусу, приоритету) и сортировку (по дате создания, дедлайну, приоритету).
* Альтернативный сценарий: Пользователь сохраняет набор фильтров как "Вид" для быстрого доступа.
* Негативный сценарий: Задачи не найдены по выбранным фильтрам, система показывает сообщение "Нет задач".
---
UC6: Редактирование задачи.
* Основной сценарий: Пользователь нажимает на задачу, выбирает "Редактировать", изменяет поля, сохраняет.
* Альтернативный сценарий: Пользователь изменяет только статус задачи через checkbox или drag&drop.
* Негативный сценарий: Пользователь пытается редактировать несуществующую задачу, система показывает ошибку.
---
UC7: Удаление задачи.
* Основной сценарий: Пользователь выбирает задачу, нажимает "Удалить", подтверждает удаление.
* Альтернативный сценарий: Пользователь удаляет несколько задач через массовые действия.
* Негативный сценарий: Задача уже удалена, система показывает сообщение.
---
UC8: Анализ продуктивности и рекоммендации.
* Основной сценарий: Пользователь переходит на вкладку "Аналитика". Система показывает графики выполненных задач, статистику по категориям, рекомендует лучшее время для работы на основе истории.
* Альтернативный сценарий: Пользователь настраивает параметры аналитики (период, категории).
* Негативный сценарий: Недостаточно данных для анализа, система предлагает начать использовать приложение активнее.
---
UC9: Интеграция с календарём.
* Основной сценарий: Пользователь настраивает интеграцию с Google Calendar. Система импортирует события и предлагает создать задачи на их основе.
* Альтернативный сценарий: Пользователь вручную синхронизирует календарь.
* Негативный сценарий: Ошибка авторизации в календаре, система просит переподключиться.
---
UC10: Напоминание о задачах.
* Основной сценарий: Пользователь устанавливает напоминание для задачи (уведомление в приложении и/или email). Система отправляет напоминание в указанное время.
* Альтернативный сценарий: Пользователь настраивает повторяющиеся напоминания.
* Негативный сценарий: Уведомления отключены в системе, система предлагает включить их.
---
UC11: Совместные задачи.
* Основной сценарий: Пользователь создаёт задачу и назначает её другому пользователю. Тот получает уведомление.
* Альтернативный сценарий: Назначенный пользователь может делегировать задачу дальше.
* Негативный сценарий: Назначенный пользователь не найден, система сообщает об ошибке.
